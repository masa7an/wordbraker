■英単語ブロック崩し
Webゲーム設計・要件定義
タイトル（仮）

Word Breaker（英単語ブロック崩し）

1. ゲームの目的とコンセプト

英単語学習を操作と結果に直結させる

正解以外は「壊れない」ことで、思考を促す

1ステージ30〜90秒の短時間プレイ

得点＋RTA（クリアタイム）で再挑戦を誘導

2. コアゲームループ
問題表示
↓
3択ブロック配置
↓
ボール発射
↓
正解ブロックのみ破壊可能
↓
全正解ブロック破壊
↓
扉解放
↓
扉にヒット → ステージクリア

3. 基本ルール定義
問題仕様

英単語 → 日本語3択

初期は単語→意味のみ

後に逆向き（意味→単語）拡張可

ブロック仕様
種類	特性
正解ブロック	1ヒットで破壊
不正解ブロック	完全無敵（音・エフェクトのみ）
装飾ブロック	背景扱い、当たり判定なし

※「壊れない」が重要。誤答を物理的に否定する。

4. 扉（ゴール）仕様

初期状態：ロック

正解ブロック全破壊 → アンロック

ボールが扉に当たると即クリア

設計意図

目的を明確に可視化

「正解→進行」の因果を強化

5. スコア設計
得点要素

正解ブロック破壊：+100

連続正解ヒット：コンボ倍率

ノーミス（ライフ減少なし）：ボーナス

RTA（リアルタイムアタック）

ステージ開始から扉ヒットまで計測

最速記録をローカル保存

ランキングは将来拡張

6. 学習効率を担保する設計（重要）
再出題制御

不正解選択肢に当てた回数を記録

ミスした単語は次ステージで再登場

新出語制限

1ステージ最大5語まで

残りは復習語

出題頻度制御（概念）
重み = ミス回数 × 未正解経過時間


※アルゴリズムはAIエージェントに委任可

7. ステージ構造
1ステージ構成

単語数：3〜5

ブロック列：3列固定

高さのみ可変

難易度段階
レベル	特徴
初級	正解ブロック密集
中級	不正解ブロック増
上級	角度が必要な配置
8. UI / UX方針
画面構成

上部：問題文

中央：ブロックエリア

下部：パドル

右上：タイマー

左上：スコア

演出方針

正解：派手（音＋光）

不正解：鈍い音のみ

解説：クリア後に1行だけ

9. 操作仕様

マウス／タッチでパドル移動

自動ボール反射（操作不要）

発射はワンクリック

10. 開発フェーズ方針（重要）

前提

第1フェーズは Python + pygame で実装する

ゲーム仕様・ロジック・難易度調整を最優先とする

このフェーズでは Web 実行・ブラウザ対応は行わない

目的は「完成度の高いゲームロジックの確立」

制約

Node.js / npm / npx / Vite は使用しない

新規 Web プロジェクトの初期化は禁止

pygame がローカルで動作することを最優先とする

補足

第1フェーズで作成した pygame コードは、後続フェーズで pygbag を用いて Web 化する前提とする

将来の Web 化を見据え、OS 依存処理・非対応 API は極力避ける



11. 技術要件（Web：最終形）

前提

ブラウザ完結

インストール不要

PC・タブレット対応（スマホ対応は後回し可）

想定構成

描画：Canvas

ロジック：JavaScript（※pygame版から移植、または pygbag 出力を利用）

データ：JSON（英単語DB）

12. 状態管理（重要）
ゲーム状態

TITLE

STAGE_START

PLAYING

STAGE_CLEAR

RESULT

単語状態

未出題

正解済

ミスあり（再出題対象）

ローカルスコア保存

14. 拡張余地（今は実装しない）

学習制御ロジックを優先

デバッグしやすい構造

まとめ（設計思想）

操作＝思考
